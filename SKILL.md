# 笔记摘要技能 (Note Summarizer)

智能分析用户转发内容，生成精炼摘要并保存到 Obsidian inbox。

## 触发模式

### 模式1：单条消息（即时整理）
```
用户：整理笔记 + [转发内容/链接/文字]
→ 立即分析并保存
```

### 模式2：上下文模式（分步整理）
```
方式A：先指令，后内容
  用户：整理笔记
  用户：[转发视频/链接/文字]  ← 自动关联，无需重复指令
  
方式B：先内容，后指令
  用户：[转发视频/链接/文字]
  用户：整理笔记  ← 自动识别上文内容
```

**上下文窗口：5分钟内，最多关联3条消息**

## 工作流程

```
检测触发词 "整理笔记" 或 "笔记摘要" 等
  ↓
判断模式：
  - 单条消息 → 立即分析内容
  - 上下文模式 → 查找关联的转发内容
  ↓
分析内容类型（文字/链接/视频/图片/混合）
  ↓
提取核心知识点
  ↓
生成精炼摘要（3-5个要点）
  ↓
智能打标签（3-5个相关标签）
  ↓
生成 Obsidian Markdown
  ↓
使用 Obsidian CLI 保存笔记
  ```bash
  obsidian create path="inbox/YYYY-MM-DD-中文标题.md" content="..." silent
  ```
  ↓
使用 Obsidian CLI 验证保存
  ```bash
  obsidian read path="inbox/YYYY-MM-DD-中文标题.md"
  ```
  ↓
回复确认 + 摘要预览
```

## 内容识别规则

### 链接类型
| 特征 | 处理方式 |
|------|---------|
| `m.toutiao.com` | 今日头条视频/文章 |
| `mp.weixin.qq.com` | 微信公众号文章 |
| `zhihu.com` | 知乎问答/文章 |
| `bilibili.com` | B站视频 |
| `youtube.com` | YouTube视频 |
| `x.com/twitter.com` | 推特/X帖子 |
| 其他URL | 通用网页抓取 |

### 内容类型判断
- **视频链接**：提取标题、UP主、核心观点
- **文章链接**：提取标题、摘要、关键段落
- **纯文字**：直接分析内容
- **图片**：描述内容，提取文字（如有OCR）
- **混合**：分别处理各部分，合并摘要

## 摘要格式

```markdown
---
title: 笔记标题
date: YYYY-MM-DD HH:mm
tags:
  - 标签1
  - 标签2
  - 标签3
source: 来源平台/作者
---

## 一句话总结
（用1句话概括核心价值）

## 核心要点
- 要点1
- 要点2
- 要点3

## Actionable 建议
- [ ] 行动项1
- [ ] 行动项2

## 原文/参考
- 链接：[标题](URL)
- 作者：@用户名
```

## Obsidian CLI 保存方法

### ⚠️ 使用前必须确认 vault 路径
```bash
obsidian vault info=path   # 先确认 vault 真实路径
obsidian files             # 确认目标文件夹名称（大小写敏感）
```

### 保存命令
```bash
obsidian create path="Inbox/2026-03-27-标题.md" content="---\ntitle: xxx\n..." silent
```

### 验证命令（必须执行）
```bash
obsidian files | grep "标题关键词"
```

### 完整工作流
1. 生成 Markdown 内容（以 YAML frontmatter 开头）
2. 使用 `obsidian create path="Inbox/..."` 保存（注意大小写与 vault 一致）
3. 使用 `obsidian files` 验证文件出现在列表中
4. **禁止用 write 工具写 Obsidian 文件** — write 需要绝对路径，容易写错位置

### ⚠️ 常见错误
| 错误 | 原因 | 正确做法 |
|------|------|---------|
| 用 write 工具写文件 | 需要猜绝对路径，容易写错 vault | 用 `obsidian create` |
| 保存后不验证 | 假设成功 | 必须用 `obsidian files` 验证 |
| 路径大小写错误 | Obsidian 索引区分大小写 | 先用 `obsidian files` 确认文件夹名 |

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 无内容可整理 | 提示"请转发或粘贴需要整理的内容" |
| 链接无法访问 | 基于标题和上下文生成摘要，标注"内容未完全获取" |
| 内容过短 | 直接原文保存，简化摘要 |
| 重复内容 | 检查inbox是否已存在，提示合并或覆盖 |

## 注意事项

1. **隐私内容**：自动识别敏感信息，添加 #private 标签
2. **待验证信息**：未经核实的内容添加 #待验证 标签
3. **临时笔记**：快速记录可简化格式，后续再整理
4. **长内容**：超过1000字自动分段提取关键信息
